Mini Compiler using LEX & YACC in C

A mini compiler project developed using LEX (Flex), YACC (Bison), and C language to demonstrate the first three phases of compiler design:

Lexical Analysis
Syntax Analysis
Semantic Analysis

The project also includes:

Error Handling
Symbol Table Management
Parse Tree Generation

📌 Features

✅ Lexical Analysis
Identifies:
Keywords
Identifiers
Numbers
Operators
Separators
Detects lexical errors (invalid tokens)

✅ Syntax Analysis
Validates input using Context-Free Grammar (CFG)
Uses LALR(1) bottom-up parsing
Generates parse tree

✅ Semantic Analysis
Detects undeclared variables
Detects duplicate declarations

✅ Symbol Table
Stores identifiers and token information

⚙️ Compiler Phases Implemented

1️⃣ Lexical Analysis

Converts source code into tokens.

Example:

int a;
a = 5;

Tokens:

int -> KEYWORD
a -> IDENTIFIER
= -> OPERATOR
5 -> NUMBER

2️⃣ Syntax Analysis

Checks whether the input follows grammar rules.

Grammar Example:

declaration → INT id_list ;
assignment  → ID ASSIGN expression ;
expression  → expression PLUS term

Parser Type:

LALR(1)
Bottom-Up Parser
Shift-Reduce Parsing

3️⃣ Semantic Analysis

Checks program meaning.

Handled Errors:

Undeclared variables
Duplicate declarations

🌳 Parse Tree Example

Input:

a = b + 5;

Output:

      a   +
         / \
        b   5

🚀 How to Run
Step 1: Generate parser files
bison -d parser.y
Step 2: Generate lexer file
flex lexer.l
Step 3: Compile
gcc lex.yy.c parser.tab.c symbol_table.c -o compiler
Step 4: Run
./compiler
